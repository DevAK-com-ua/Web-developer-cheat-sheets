Git — Налаштування

Інсталяція, налаштування, початок роботи

git config --global user.name "Ім'я"
— Задає ім'я користувача для всіх репозиторіїв.
git config --global user.email "email@example.com"
— Задає email для комітів.
//Вказані ім'я та email людини відображається у змінах, та дає змогу бачити хто їх вносив.//
git config --list
— Показує всі налаштування git.
git init
— Створює новий локальний git-репозиторій у поточній папці (git починає відстежувати папку).

Git — Основні команди

Додавання, фіксація, перегляд змін

git status
— Показує статус репозиторії, чи є файли які не відстежуються, або були змінені.
git add .
— Додає всі зміни у відстеження (staging area) перед комітом.
//Замість крапки можна вказати назву окремого файлу, якщо в staging area потрібно передати лише його.//
git commit -m "повідомлення"
— Створює коміт, зберігаємо та коментуємо зміни.
git log
— Виводить історію комітів.
git log --oneline
— Коротка версія історії комітів.
git log -p
— Показує зміни для кожного коміту.
//Історія - розширено, щоб вийти з цього режиму q.//

Git — Перегляд змін

Відмінності та деталі комітів

git diff
— Показує відмінності незакомічених зміни від останнього коміту.
git show
— Показує деталі змін останнього коміту.
git show #
— (#-тег коміту) Показує деталі змін вказаного коміту.

Git — Редагування та скасування змін

Робота з версіями файлів, комітів

git commit --amend --no-edit
— Вносить зміни в останній коміт без зміни коментаря.
git restore --staged шлях/файл
— Повертаємо файл зі staging area.
git restore шлях/файл
— Скасовує зміни у файлі, повертає останню версію файлу з останнього коміту.
git restore --source=# шлях/файл
— (#- тег коміту) Повертає вказанмй файл до конкретного, вказаного коміту.
git rm шлях/файл
— Видаляє файл з робочої директорії та з staging area і додає це видалення до коміту.
//Щоб повернути цей файл, потрібно використати
git restore --staged шлях/файл
, в такому варіанті git поверне видалений файл.//
//Щоб замість видаленого файлу повернути старішу версію, потрібно використати
git restore --source=# шлях/файл
.//
git reset --soft #
— (#- тег коміту) Відкат до вказаної версії коміту, зберігаючи зміни у staging area.
git reset --hard #
— (#- тег коміту) Відкат до вказаної версії коміту, із видаленням наступних комітів повністю.

Git — Очищення проєкту

Видалення файлів

git clean -n
— Показує кандидатів на видалено.
git clean -f
— Видаляє непроіндексовані файли (не відслідковувані Git).

Git — Робота з гілками

Створення, видалення, переміщення

git branch
— Показує список гілок та в якій ми знаходимось.
git branch name-branch
— Створює нову гілку.
git checkout name-branch
— Перемикається на вказану гілку.
git checkout -b name-branch
— Створює і перемикається на нову гілку. Альтернатива:
git branch -b name-branch
.
git branch -d branch-name
— Мʼяке видалення гілки, видаляє якщо гілка вже злита з основною (запобігає втраті змін).
git branch -D branch-name
— Жорстке видалення гілки, всі зміни які були внесені та не злиті в основну гілку будуть видалені.

Git — Злиття гілок

Обʼєднання, застосування змін

git merge branch-name
— Виконується з main-гілки та обʼєднуєднує-застосовує зміни з вказаної гілки.
//Можна використовувати не лише в main, тоді зміни-злиття буде застосовано до поточної гілки.//

Git — Конфлікти та їх вирішення

Коли виникає конфлікт?

  • Дві гілки змінюють один і той самий рядок файлу.
  • Одна гілка видаляє файл, а інша — змінює його.
  • Обидві гілки додають однакові імена файлів у різних версіях.

Як виправити?

//Після злиття, якщо виник конфлікт — Git позначає конфліктні місця у файлах.//
//Виправ конфлікт вручну в коді, потім://
git add файл
— Додає виправлений файл у staging area.
git commit
— Завершує злиття.

Git — Робота з GitHub

Синхронізація локального репозиторію з GitHub

git remote add origin URL
— Додає віддалений репозиторій (один раз після git init).
git push -u origin main
— Вперше надсилає локальну гілку на GitHub і встановлює її як основну.
git push
— Надсилає нові коміти у вже пов’язаний віддалений репозиторій (після первинного -u).
git pull
— Завантажує та застосовує останні зміни з GitHub до поточної гілки.
git fetch
— Завантажує останні зміни з GitHub без автоматичного злиття з поточною гілкою (на відміну від pull).
git remote -v
— Показує список віддалених репозиторіїв (часто використовують для перевірки, до якого GitHub-репозиторію підключено локальний проєкт).
git push origin branch-name
— Відправка будь-якої гілки, не лише main.
git pull origin branch-name
— Отримання змін з конкретної гілки.

Клонування репозиторію

git clone URL
— Копіює віддалений репозиторій з GitHub на локальний комп’ютер (створює локальну копію).
